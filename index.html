核心交互流程
1
首次进入 — A方案气泡引导
输入框上方展示3个推荐词条气泡，8秒无操作自动消失
2
点击词条 → 填入输入框
词条填入可编辑，用户可修改后发送；发送后切换至轮播状态
3
激活后 — D方案 Placeholder 轮播
输入框 placeholder 每3秒换一条推荐词，点击自动填入
4
输入中自动暂停，发送后刷新词条
用户打字时不打断，消费词条或ASR内容变化后触发刷新
三层词条生成架构
Layer 1 · 场次词库（最高优先级）
运营针对单场直播手动配置，如抽奖→"我要参与"
Layer 2 · 全局词库（兜底）
平台维护通用词条，如"好看""链接放哪""主播加油"
Layer 3 · LLM 动态生成（主力）
基于 ASR 文本 + 弹幕流实时生成，倾向多用 LLM
关键设计决策
触发时机
ASR + 弹幕流自动触发
状态切换
首次发弹幕后切换
词条数量
每次展示 3 个
轮播间隔
3秒（运营可配置）
气泡消失
8秒超时或首次发言
个性化
身份分层叠加专属词
